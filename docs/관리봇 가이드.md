# 관리봇 가이드 (v1.1.0)

모든 봇을 관리하고 모니터링하기 위한 오너 전용 유틸리티 봇입니다.

## 권한 모델

| 등급 | 대상 | 범위 |
|------|------|------|
| **오너** | 봇 운영자 1명 (방마다 해시 다름) | 관리봇 모든 명령어 |
| **일반** | 나머지 | 공개 명령어만 (핑, 방정보, 내정보, 관리도움) |

> 관리봇은 **오너 전용**입니다. 다른 봇(채팅봇 등)에서 일반 유저에게 권한을 부여하는 "어드민"과는 별개입니다.

### 오너 해시 관리

카카오톡 유저 해시는 방마다 다르기 때문에, 오너 해시는 두 곳에서 병합 관리됩니다:

1. **하드코딩** (`OWNER_HASHES_DEFAULT`) — 코드에 직접 기록된 기본 해시 목록
2. **파일** (`sdcard/bot/owner_hashes.json`) — `!오너해시추가` 명령으로 동적 추가

새 방에서 오너 인증이 필요할 때:
1. 이미 등록된 해시가 있는 방에서 오너로 인증된 상태 확인
2. 새 방에서 `!오너해시추가` 실행 → 현재 방의 해시가 파일에 저장됨

## 명령어 목록

### 공개 명령어

| 명령어 | 설명 |
|--------|------|
| `!핑` / `!테스트` | 봇 생존 확인 (퐁! 응답) |
| `!여기` / `!방정보` | 현재 방 이름, 채널 ID, 그룹채팅 여부, Log ID |
| `!나` / `!내정보` | 본인 이름, 유저 해시, 오너 여부 |
| `!관리도움` / `!관리명령어` | 명령어 도움말 (오너면 전체, 일반이면 공개만 표시) |

### 모니터링 (오너)

| 명령어 | 설명 |
|--------|------|
| `!봇목록` | 설치된 전체 봇 + 전원 상태 (🟢/🔴) |
| `!응답체크` | 현재 방 기준 봇별 상태 (정상/미적용/세션없음/전원꺼짐) |
| `!봇상태` | 종합 대시보드 (전원+응답+적용 상태, 업타임, 메모리) |
| `!업타임` | 관리봇 업타임 표시 |
| `!업타임 <봇이름>` | 해당 봇의 전원 상태 + 관리봇 업타임 |
| `!메모리` | JVM 힙 메모리 사용량 (사용/할당/최대/여유/사용률) |

### 봇 제어 (오너)

| 명령어 | 설명 |
|--------|------|
| `!봇켜기 <이름>` | 봇 전원 ON (`setPower(true)`) |
| `!봇끄기 <이름>` | 봇 전원 OFF (`setPower(false)`) |
| `!컴파일 <이름>` | 봇 재컴파일 (`compile()`) |
| `!로그 [봇이름] [줄수]` | 최근 로그 조회 (기본 관리봇 10줄, 최대 30줄) |

### 방별 적용 관리 (오너)

| 명령어 | 설명 |
|--------|------|
| `!적용설정` | 현재 설정 조회 (봇별 적용된 방 목록) |
| `!적용설정 초기화` | 현재 봇 목록 기반으로 설정 파일 생성 |
| `!적용설정 추가 <봇이름>` | **현재 방**에 해당 봇 적용 등록 |
| `!적용설정 제거 <봇이름>` | **현재 방**에서 해당 봇 적용 해제 |

> 방 이름을 직접 입력할 필요 없이, **해당 방에서 명령어를 실행하면 channelId가 자동 사용**됩니다.

### 유틸 (오너)

| 명령어 | 설명 |
|--------|------|
| `!해시검색 <닉네임>` | 현재 방에서 해당 닉네임의 유저 해시 조회 (comm_db 기반) |
| `!권한테스트` | 파일 쓰기/읽기/삭제 권한 확인 |
| `!객체확인` / `!덤프` | CMD 객체 속성 타입 덤프 |
| `!오너해시추가` | 현재 방의 해시를 오너 목록에 동적 추가 |

## 설정 파일

| 파일 | 용도 |
|------|------|
| `sdcard/bot/owner_hashes.json` | 동적으로 추가된 오너 해시 목록 (`string[]`) |
| `sdcard/bot/bot_rooms.json` | 방별 봇 적용 설정 (channelId 기반) |

### bot_rooms.json 구조

```json
{
  "마비노기봇": ["18448842407409693", "12345678901234567"],
  "채팅봇": ["18448842407409693"],
  "_roomNames": {
    "18448842407409693": "지통실",
    "12345678901234567": "수다방"
  }
}
```

- 봇 이름을 key로, 적용할 방의 `channelId` 배열을 value로 저장
- `_roomNames`: channelId → 방 이름 매핑 (조회 시 표시용, 자동 갱신)
- 빈 배열 `[]`: 모든 방에서 동작
- 봇이 등록되지 않은 경우: 전체 방 적용으로 간주

## 응답체크 상태 아이콘

| 아이콘 | 상태 | 의미 |
|--------|------|------|
| 🟢 | 정상 동작 | 전원 ON + 이 방에 적용됨 + 세션 있음 |
| 🟡 | 세션 없음 | 전원 ON + 적용됨이나 채팅 기록 없음 |
| ⬜ | 미적용 | 전원 ON이지만 이 방에서 동작하지 않음 |
| 🔴 | 전원 꺼짐 | 봇 자체가 비활성 |

## 초기 설정 순서

1. 관리봇 컴파일 및 활성화
2. 아무 방에서 `!내정보`로 오너 해시 확인
3. `!적용설정 초기화` — 봇 목록 기반 설정 생성
4. 각 봇이 동작해야 할 방에서 `!적용설정 추가 <봇이름>` 실행
5. `!응답체크`로 결과 확인
